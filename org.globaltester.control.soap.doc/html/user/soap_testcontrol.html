Remote test case execution

The remote control SOAP service consists of two different published endpoints. The SOAP endpoint for controlling the test execution in the GlobalTester presents the available methods and data types.

To allow bidirectional communication, the callback must be available as a SOAP endpoint itself. The WSDL for this can be found at the URL http://host:port/globaltester/DummyControl?wsdl as described here. Allowed Objects that can be transfered via the given methods are described in the schema available at http://host:port/globaltester/DummyCallback?xsd=1. The published sample callback can be used for the test control interface with the following behaviour:

  • Dismiss/Ignore all notifications
  • Choose the first available answer to all user entry prompts

The test case results, that are returned after the test cases, are encoded in integer values as follows:

PASSED0
WARNING2
FAILURE1
REQUIREMENT_MISSING5
UNDEFINED4
NOT_APPLICABLE3

For reference there are implementations in the com.secunet.globaltester.testcontrol.soap.consumer and com.secunet.globaltester.testcontrol.callback.soap.provider bundles.
